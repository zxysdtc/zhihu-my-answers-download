# 知乎「用户回答」增量下载（油猴脚本）

一个 Tampermonkey 油猴脚本：下载**任意知乎用户**的回答为 Markdown 文件。
面板内输入（或自动识别正在浏览的）目标用户，点「选择文件夹」弹出**原生目录选择器**，
文件**直接写入你选的目录**；**增量更新**——只下新增/有改动的，按用户分别记录。
**无任何外部依赖库**（HTML→Markdown 内置实现）。

## 功能

- 下载任意用户的回答：面板输入 `url_token` 或粘贴主页链接；浏览某人主页时自动填好
- 原生文件夹选择器（File System Access API），文件直接写入所选目录，更新的回答原地覆盖
- 浏览器不支持文件夹选择器时，自动回退到 `浏览器默认下载目录/zhihu-answers/`
- 增量：按用户分别记录「回答 ID + 更新时间」，仅下载新增或被编辑过的回答
- 每条回答单独存为 `用户名_问题标题_回答ID.md`
- 内置轻量 HTML→Markdown（标题/加粗/链接/图片/列表/引用/代码），无需联网
- 油猴菜单提供「清空全部增量记录（重新下载）」

## 安装

1. 安装 [Tampermonkey](https://www.tampermonkey.net/) 扩展
2. 新版 Chrome 需在 `chrome://extensions` → Tampermonkey 详情中打开 **「允许用户脚本 / Allow user scripts」**
3. 打开 [脚本 raw 链接](https://github.com/zxysdtc/zhihu-my-answers-download/raw/main/zhihu-my-answers-download.user.js) 安装
4. 打开知乎任意页面，右下角出现下载面板

## 使用

1. 在面板输入框填目标用户的 `url_token`（如 `zhang-san`）或直接粘贴其主页链接；
   若你正停留在某人主页，会自动填好
2. 点「选择文件夹」，在弹出的系统对话框中选择保存目录（可选坚果云同步盘内的目录）
3. 点「下载」，开始增量同步；再次点击只会下载新增/改动的回答

> 提示：`url_token` 就是用户主页链接 `zhihu.com/people/<这一段>` 中的那一段。

## 配置（脚本顶部 CONFIG）

| 项 | 说明 | 默认 |
| --- | --- | --- |
| `fallbackSubDir` | 不支持目录选择器时的回退子目录 | `zhihu-answers` |
| `pageLimit` | 每页拉取数量（知乎上限 20） | `20` |
| `pageDelayMs` | 翻页延迟（毫秒） | `700` |
| `writeDelayMs` | 文件之间的写入间隔（毫秒） | `120` |

## 保存到坚果云

直接在「选择文件夹」时选中**坚果云同步盘内的目录**即可，文件写入后自动同步上云，
无需改浏览器下载目录。若浏览器不支持目录选择器走回退方案，再按下面设置：

1. Chrome 设置 → 下载内容 → 将「下载位置」设为坚果云同步盘下的文件夹，关闭「下载前询问位置」
2. Tampermonkey 设置 → 下载模式选「浏览器 API」

## 限制说明

- 文件夹选择器需 Chromium 内核浏览器（Chrome/Edge）且 HTTPS 环境（知乎满足）；
  Firefox 等不支持时自动回退到下载目录方式
- 出于浏览器安全限制，选择目录/授权必须由用户点击触发，无法静默进行
- 仅访问知乎自身接口，运行在真实登录上下文中；下载非公开/被限制内容仍受知乎权限约束
